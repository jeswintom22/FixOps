# Incident Memory Design Task

Do not write code.

Design the Incident Memory architecture.

Context:

FixOps follows:

Detect
→ Investigate
→ Decide
→ Act
→ Verify
→ Learn

The Incident Memory layer must support future Context Engineering and Multi-Agent workflows.

Tasks:

1. Define the Incident entity.

2. Define all related entities.

3. Define relationships between:

   * Incident
   * Findings
   * Recommendations
   * Actions
   * Verification Results
   * Knowledge Entries

4. Define what information should be embedded.

5. Define when memory should be created.

6. Define retrieval strategy.

7. Define how memory becomes context for agents.

8. Define IncidentContext object.

Output:

* Entity diagram
* Data flow
* Context flow
* Retrieval flow

Do not write implementation code.

Focus only on architecture.
